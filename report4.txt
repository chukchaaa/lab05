# Лабораторная работа 04
## Тема
Изучение систем непрерывной интеграции на примере GitHub Actions.



## Цель работы
Настроить автоматическую сборку проекта с использованием GitHub Actions и добавить статусный значок в README.


## Ход выполнения

### 1. Подготовка репозитория

Создаем репозиторий lab04 на основе lab03:

```bash
git clone https://github.com/${GITHUB_USERNAME}/lab03.git projects/lab04
cd projects/lab04
git remote remove origin
git remote add origin https://github.com/${GITHUB_USERNAME}/lab04.git
```
### 2. Создание GitHub Actions workflow
Создаем файл .github/workflows/ci.yml со следующим содержанием:

```yaml
name: CI

on:
  push:
    branches: [ "main", "master" ]
  pull_request:
    branches: [ "main", "master" ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v4

    - name: Configure CMake
      run: cmake -H. -B_build -DCMAKE_INSTALL_PREFIX=_install

    - name: Build
      run: cmake --build _build

    - name: Install
      run: cmake --build _build --target install

    - name: Run examples
      run: |
        _build/example1
        _build/example2
        cat log.txt
```

### 3. Запуск CI
После пуша workflow автоматически запустился на GitHub:

```bash
git add .github/workflows/ci.yml
git commit -m "add GitHub Actions CI workflow"
git push origin main
```
Сборка прошла успешно

### 4. Добавление статусного значка
В файл README.md добавлен значок статуса CI:

```markdown
## CI Status

[![CI](https://github.com/chukchaaa/lab04/actions/workflows/ci.yml/badge.svg)](https://github.com/chukchaaa/lab04/actions/workflows/ci.yml)
```
### 5. Проверка статуса
Через терминал:

```bash
gh run list
gh run view --log
```
